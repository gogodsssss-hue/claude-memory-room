# 텔레그램 방 / 봇 세팅 — 실수와 교훈 (2026-06-01)

다음에 같은 작업을 할 때 바로 쓰기 위한 기록. (비밀키·토큰·비번은 절대 적지 않는다.)

## 1. 가장 큰 실수 — 방에 "엉뚱한 봇"을 넣었다
- 봇이 둘이다: `@Irsar_bot`(코드 telegram_listener.py를 실제로 돌리는 봇) / `@irir4r_secretary_bot`("비서 영숙", 코드 안 돌림).
- 코드는 `.env`의 `TELEGRAM_BOT_TOKEN`을 쓰고, 그 토큰 주인은 `@Irsar_bot`이다.
- 새 방엔 "비서 영숙"을 넣어서 → 코드 돌리는 봇이 방에 없으니 chat_id도 메시지도 안 잡혔다.
- 교훈: 방에 넣을 봇 = "코드가 쓰는 토큰의 봇". 헷갈리면 `getMe`로 토큰 주인부터 확인.

## 2. 그룹 chat_id 잡는 법
- 봇을 그룹에 추가/관리자 지정하는 순간 `my_chat_member` 이벤트 발생(프라이버시 무관) → 직후 `getUpdates`로 확인.
- 그룹 chat_id는 마이너스(-100…)로 시작. 잡으면 `.env`에 `NEWS_ROOM_CHAT_ID=<번호>` 저장.
- 교훈: getUpdates가 개인 채팅만 보이면 → 토큰이 "그 방에 없는 봇" 것이다(1번과 같은 원인).

## 3. Group Privacy
- 그룹에서 "뉴스 …" 같은 일반 메시지를 읽으려면: BotFather에서 Privacy Disable, 또는 봇을 관리자로.
- 실수: 프라이버시를 엉뚱한 봇(영숙) 걸 껐다. 정작 켤 건 코드 봇 `@Irsar_bot`.

## 4. 봇 안전 재시작
- pkill로 run_bot_forever.sh / telegram_listener.py 둘 다 종료 → getUpdates?offset=-1로 큐 비움 → nohup 재기동 → "감시자 1 / 리스너 1" 확인.

## 5. 보안
- 봇 토큰이 스크린샷에 노출된 적 있음 → 공개 금지, 의심 시 BotFather 재발급.
- `.env`, `*.bak*`는 깃허브에 절대 올리지 않는다(.gitignore 차단).

## 6. 방 분리 설계 메모
- 뉴스 방: `뉴스 [주제]` → 미미 초안 → WP 임시저장.
- (예정) 드라마·영화 방: `드라마 [제목]`/`영화 [제목]`, 공개된 사실만·루머 금지.
- 출력은 명령 보낸 방으로 자동 회신되므로 방마다 트리거만 다르게 쓰면 동작이 갈린다.
