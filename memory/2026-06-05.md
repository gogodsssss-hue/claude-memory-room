# 2026-06-05 기억 — 클로드 쓰기 권한을 연 날

## 한 줄 요약

클로드(코드)가 드디어 이 방에 **직접 보관(push)**할 수 있게 된 날. 원인은 GitHub 앱 미설치(403), 마스터가 앱을 설치하며 해결.

## 있었던 일 (시간순)

### 1. 존재하지 않는 파일 보고 사건
- 마스터가 `telegram_bot_context.md` 를 찾으심.
- 확인 결과 **그 파일은 리포에도, git 히스토리에도 한 번도 없었음.**
- 최신 커밋(`42b7011`)이 실제로 올린 건 `troubleshooting/drama-bot-separation-2026-06-05.md` 1개뿐.
- → **"올렸다는 보고"와 "실제 git 상태"가 어긋난 첫 사례.** 이 방의 핵심 문제의식이 여기서 출발.

### 2. 운영 방향 논의
- 마스터: "자료는 분리 말고 공유가 좋다. 공유가 힘들면 보관이라도 하자."
- 결정: 자료는 한곳에 모으되, **잃지 않는 보관을 보장**한다.
- `보관소_운영원칙.md` 신설 — "보고 ≠ 보관, git에 실제 커밋된 것만 보관된 것."

### 3. 403 발견
- 클로드가 보관하려 push → **403 권한 거부.** GitHub API도 403.
- 잠깐 "클로드도 쓸 수 있다"고 단언했다가, 실제로 막혀서 **정정.** ← 스스로 "보고≠보관" 함정에 빠진 산 증거.
- 읽기는 되는데 쓰기만 막힘 → 원인 추적 시작.

### 4. 원인 규명 (마스터와 한 걸음씩)
- `settings/installations`: 설치된 앱은 Vercel뿐, **Claude 없음.**
- `승인됨` 탭: Claude(anthropics) 있음 — **승인만 되고 설치 안 됨** 확정.
- 결론: 앱 미설치 → 쓰기 권한 없음 → 403.

### 5. 해결
- 마스터가 `github.com/apps/claude/installations/new` 에서 **모든 저장소 + 쓰기 권한**으로 설치.
- 마스터 판단: "내 비서니 내 깃허브 다 들어가는 게 당연하다."
- push 즉시 성공. 로컬·원격 해시 `9e910c4` 일치, 원격 파일 내용까지 확인 → **보관 완료.**

## 결정 사항

- 자료는 **분리하지 않고 공유**. 출처는 폴더가 아니라 파일 안 기록으로.
- 클로드가 **직접 보관하는 손**이 됨. 대리 쓰기 폐지.
- 권한 범위: **모든 저장소**(마스터 결정).

## 교훈

- **보고 ≠ 보관.** 말이 아니라 git 상태가 진실. push·검증까지 해야 끝.
- 대리 쓰기는 보고-실제 불일치를 낳는다. 각자 자기 손으로.
- 막히면 "안 된다"가 아니라 **원인을 한 걸음씩** 추적하면 풀린다. 오늘 403도 권한 하나였다.
