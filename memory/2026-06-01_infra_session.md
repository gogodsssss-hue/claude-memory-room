## 2. 트러블슈팅 (오늘 겪은 사고)

### 사고 A — 미미 무응답 (NameError)
- **증상**: "미미야 …" 보내면 무응답(에러 메시지조차 없음). 라우팅 자체는 정상 도달.
- **원인**: `handle_mimi_command`(및 `handle_codex_command`)가 파일에서 `if __name__ == "__main__": main()` **줄보다 아래**에 정의됨. main()이 먼저 실행되어 블로킹 → 아래 함수 정의가 실행(바인딩)되지 못함 → 호출 시 `NameError: name 'handle_mimi_command' is not defined` → 메인 루프 `except`가 조용히 삼킴 → 무응답.
- **해결**: `if __name__ == "__main__": main()` 블록을 **파일 맨 끝으로 이동**. 모든 함수가 먼저 로드된 뒤 main 실행.
- **교훈**: 핸들러 `def`는 반드시 `main()` 호출보다 위. 정석은 `if __name__` 블록을 파일 최하단에 두기. (NameError 무응답의 단골 원인. 메인 루프의 `traceback.print_exc()` 덕에 로그에서 즉시 진단됨.)

### 사고 B — 터미널 붙여넣기 사고 (반복)
- **증상**: ① 파이썬 코드를 터미널 명령창에 직접 붙여넣어 `syntax error` 무더기. ② 긴 heredoc 붙여넣기가 중간에 잘려 깨지거나 `>`에서 멈춤.
- **해결/교훈**:
  - 코드는 "명령창"이 아니라 "파일"에 들어가야 함 → 자동 삽입/교체 스크립트(python)로 처리.
  - 긴 코드는 (a) 짧게 쪼개거나, (b) **base64 한 줄**로 인코딩 후 `echo '...' | base64 -d > 파일` 로 넣으면 안 깨짐(줄 단위 스크램블·heredoc 종료 문제 없음).
  - 파일 수정은 항상 **백업 + `ast.parse` 문법검사 통과 후 저장**. (생성된 백업: `.bak_mimi`, `.bak_mimi2`, `.bak_claude`, `.bak_order`)
