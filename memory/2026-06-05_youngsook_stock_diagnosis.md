## 1순위 의심 원인 — 주빵이 모듈 경로 불일치

`youngsook_listener.py` 안에서 주빵이(stock_expert) 위치를 **두 군데가 다르게 적음**:

- **모듈 import (40번 줄):** `HERE/../../../agents/stock_expert/skills` — `_company` **없음**, 3단계 상위
- **persona (155번 줄):** `HERE/../../../agents/stock_expert/prompt.md` — 동일
- **watchlist (739번 줄):** `HERE/../../../../_company/agents/stock_expert/watchlist.csv` — `_company` **있음**, 4단계 상위

→ 둘 중 하나는 틀린 주소. 만약 **40번(import)이 틀리면** `juppang_analyzer = None` →
주식 요청마다 **"juppang_analyzer 모듈을 불러올 수 없습니다"** 만 뱉고 끝남(911~1154번 분기).
**이게 "주식 안 써짐"의 가장 유력한 원인.**

> 확정하려면 실제 폴더 트리(어디에 `stock_expert`가 있는지)와 영숙이 기동 로그가 필요. 업로드받은 건 단일 `.py` 파일뿐이라 경로 검증 불가.

## 조치 (1차 수정 파일 제작·전달)

증상 확정 전이지만, **1순위 의심 원인(경로 불일치)은 손해 없이 선제 수정 가능**하다고 판단해 고친 파일을 제작·전달함.

- **수정 내용:** `juppang_analyzer` 불러오는 부분(39~44번 줄)을 **단일 경로 → 여러 후보 경로 자동 탐색**으로 교체.
  - `_company` 유무, 상위 깊이(3~4단계) 등 7개 후보를 돌며 `juppang_analyzer.py`가 실제 존재하는 폴더를 찾아 import.
  - 못 찾으면 확인한 후보 경로 전부를 로그로 출력(다음 진단 단서).
- **효과:** 경로 어긋남이 원인이었다면 → 주식 분석 즉시 복구. 아니어도 부작용 없음(방어적 추가).
- **검증:** `python3 -m py_compile` 문법 통과. (실제 작동 테스트는 마스터 서버 배포 후 가능)
