# 보관소 운영 원칙

> 이 방은 **자료를 분리하지 않고 한곳에 모으는 공유 기억 방**이다.
> 실시간 공유가 힘들 때라도, **잃어버리지 않게 보관하는 것**만은 반드시 보장한다.

## 1. 한 줄 원칙

**보고 ≠ 보관.** "올렸다"는 말이 아니라, **git에 실제로 커밋된 것만** 보관된 것이다.

## 2. 왜 이 원칙이 필요한가

예전 구조는 "쓰기는 서버 에이전트, 클로드는 읽기만"이었다.
그래서 한쪽이 다른 쪽 몫까지 **대리로 써주는** 일이 생겼고,
그 과정에서 **"올렸다고 보고했는데 실제 파일은 없는"** 불일치가 반복됐다.
(예: 2026-06-05, 존재하지 않는 `telegram_bot_context.md`를 올렸다고 보고된 사건)

대리 쓰기는 폐지한다. 자료를 만든 주체가, 또는 **확실히 보관할 수 있는 손**이 직접 보관한다.

## 3. 보관하는 손

- **클로드(Claude Code)는 이제 직접 커밋·푸시할 수 있다.** 읽기 전용이 아니다.
- 따라서 공유가 막히거나 저쪽(안티그래비티/코다리)이 못 올리는 상황이어도,
  자료만 넘겨주면 클로드가 **이 방에 확실히 보관**한다.
- 보관의 손은 한 명이라도 확실하면 된다. 둘이 헷갈리게 쓰지 않는다.

## 4. 자료는 합친다 (분리하지 않는다)

- 누가 만든 자료든 **같은 저장소 안에** 둔다. 사람·페르소나별로 칸을 쪼개지 않는다.
- 출처는 폴더 분리가 아니라 **파일 안 기록**으로 남긴다 (누가/언제 만든 자료인지 한 줄).

## 5. 보관 확인 규칙

자료를 보관했다고 말하기 전에, 반드시 다음을 확인한다.

1. `git status` — 워킹트리에 실제로 파일이 있는가
2. `git log` / `git show` — 그 파일이 실제 커밋에 포함됐는가
3. 푸시까지 끝났는가 (원격에 올라갔는가)

세 가지가 모두 확인돼야 "보관 완료"다. 그 전까지는 "보관 중"이다.

## 6. 변하지 않는 금지선

- 토큰·키·비밀번호·개인정보는 절대 보관하지 않는다. (기존 README 규칙 유지)
