# claude-memory-room

> 클로드(Claude)가 텔레그램 봇(comodo-ingong)을 다루며 겪은 일 — 실수, 성공, 진단 과정 — 을 기록하는 공개 기억 방.
> 다음에 같은 문제가 또 생겼을 때 처음부터 헤매지 않기 위한 누적 기록.

## 이 방의 목적
- 텔레그램 봇/에이전트(주빵이·코덱스·미미 등) 작업 중 무엇이 안 됐고, 어떻게 고쳤는지를 남긴다.
- 공개(Public)로 두어, 클로드(및 허락한 다른 AI)가 URL만으로 읽고 참고할 수 있게 한다.
- 비밀(토큰·키·비공개 코드)은 절대 여기 올리지 않는다. 그런 건 비공개 레포(comodo-ingong)와 서버 .env에만 둔다.

## 구조
- README.md : 이 파일
- troubleshooting/telegram-bot.md : 진단·해결 런북 (재사용용 플레이북)
- troubleshooting/multi-room-setup.md : 다중 방(뉴스·드라마·영화) 개설 런북
- memory/YYYY-MM-DD.md : 날짜별 기억 (실수·성공 로그)
- 보관소_운영원칙.md : 공유·보관 운영 원칙 (보고≠보관, 보관 확인 규칙)

## 사용 방식
- 자료는 분리하지 않고 한곳에 모은다 (공유 우선). 공유가 힘들어도 보관은 보장한다.
- 쓰기/보관: 서버 에이전트뿐 아니라 **클로드(Claude Code)도 직접 커밋·푸시**할 수 있다. 더 이상 읽기 전용이 아니다.
- 읽기(참고): 클로드가 raw URL로 읽는다.
- 자세한 운영 원칙은 보관소_운영원칙.md 참고. ("보고 ≠ 보관" — git에 실제 커밋된 것만 보관된 것)

## 기록 규칙
- 한 사건 = 한 항목. 증상 → 원인 → 해결 → 교훈 순서로.
- 비밀번호/토큰/키/개인정보 금지.
- 반복되는 문제는 troubleshooting/ 의 런북으로 승격해 정리.
