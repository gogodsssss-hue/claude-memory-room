# 드라마·영화방 대화형 블로그 파이프라인 — 사양서 (2026-06-06)

> 목적: 드라마방을 **드라마·영화·신작 등 최신작 블로그 작성 방**으로 구현. 뉴스방과 같은 파이프라인, 주제만 연예·영상물.
> 짝 문서: `news-room-pipeline-spec-2026-06-06.md` (기본 파이프라인·네이버 수집 코드 동일). 이 문서는 **드라마 차이점(delta)만** 정리.

## 1. 마스터 요구 (확정)
- 드라마방 = **블로그 글 쓰는 방** (드라마·영화·신작 등 최신작).
- 뉴스방처럼 **대화형** + **네이버 수집 우선**.
- 파이프라인 동일: **코덱스(네이버 수집) → 데스크(주제 선정) → 미미(글) → 루나(이미지) → 워드프레스 임시저장**.

## 2. 뉴스방과 같은 부분 (재사용)
- `naver_news_search()` 공용 함수로 수집(네이버 1순위 → 구글 RSS → Tavily 폴백). 코드는 news 사양서 3장 참고.
- `handle_mimi_command(prefetched_facts=...)` 실제 사용하도록 수정(현재 미사용) → 코덱스가 모은 자료로 미미가 글 쓰게.
- 단계별 진행 멘트(대화형), 사용자 주제·수정 요청 반영.
- 현재 코드: `handle_drama_command(query, ...)` 가 이미 코덱스 수집 → 미미 작성 흐름. 단 **수집이 `extract_search_keywords + google_news_search`** → **네이버로 교체** 필요.

## 3. 드라마 전용 차이 (delta)

### 3-1. 수집 검색어
- 주제에 "드라마/영화" 맥락을 붙여 검색: 예) `naver_news_search(f"{topic} 드라마 영화")` 또는 작품명 기반.
- 트리거: `드라마 [제목]`, `영화 [제목]`, `신작 [제목]` (youngsook_listener 557~564번 줄에 이미 존재).

### 3-2. 연예 가드레일 (★중요 — 명예훼손·루머 방지)
미미 프롬프트에 반드시 포함 (multi-room-setup.md 3장 기준):
- **공개된 사실만.** 출연진·줄거리·공개일 등은 **공식 발표·공인 매체** 기준만.
- **루머·미확인 열애설·사생활 추측 금지.**
- 인물 명예훼손·확정적 단정("~한 게 분명하다") 금지 → 불확실하면 `[확인 필요]`.
- **스포일러는 "스포일러 주의" 경고 뒤에만.**

### 3-3. 팩트체크 게이트 (드라마방 고질병 — 기존 진단 반영)
- 기존 문제(진단: `drama-bot-factcheck-gate-diagnosis-2026-06-05.md`): **코덱스가 검색결과 5건을 날것(raw)으로 미미에게 던져** 미미가 사실 아닌 내용을 쓸 위험.
- 해결: 코덱스 단계에서 **팩트시트(정리된 검증 자료)** 형태로 가공해 미미에 전달.
  - 코덱스 출력 = 항목별 "확인된 사실 / 출처 / 미확인" 분류. raw 덤프 금지.
  - 미미는 **팩트시트에 '확인된 사실'로 분류된 것만** 본문 근거로 사용, 미확인은 `[확인 필요]`.
- (선택) 코덱스 페르소나 파일 신설: "검증관 + 팩트시트 출력" 역할. (chat 클로드가 제시했던 방향)

## 4. 대화형 흐름 (드라마방)
1. 사용자: `드라마 [제목]` / `영화 [제목]` / 또는 주제
2. 코덱스: `naver_news_search(제목+드라마/영화)` → "🎬 코덱스 | 네이버 N건 수집" → 팩트시트로 정리
3. 데스크: 글감/앵글 선정 멘트 (사용자 다른 각도 원하면 교체)
4. 미미: 팩트시트(확인된 사실)로 **연예 가드레일 적용** 블로그 초안 작성 → 방에 표시
5. 루나: 썸네일 → 워드프레스 임시저장 → "📝 임시저장 완료 + 링크"
6. 사용자 "고쳐/다른 작품" → 해당 단계만 재실행

## 5. 배포·주의
- **comodo 클로드가 구현·테스트·배포** (워드프레스·루나·네이버키 서버에서만 가능).
- 드라마봇은 이미 분리됨(`drama_listener.py`, DRAMA_BOT) → 거기에 네이버 수집·팩트시트·연예가드레일 적용.
- 본체(youngsook_listener) 공용 함수(`naver_news_search`, `handle_mimi_command` prefetched_facts) 수정은 뉴스·드라마 공통 → **한 번에 정리, 영숙 작업과 순차로**(충돌 방지).
- 🔒 키는 서버 .env에만.
