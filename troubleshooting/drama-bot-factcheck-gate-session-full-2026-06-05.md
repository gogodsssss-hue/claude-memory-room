## 0. 등장 요소

- **미미:** 콘텐츠 에디터 에이전트. 글만 쓴다. (페르소나: `personas/mimi_persona.md`)
- **코덱스:** 팩트체크 담당으로 설계됨. 현재 전용 페르소나 파일 없음.
- **코다리:** 시스템 총괄 집행관. 로컬 시스템 관장.
- **스승(클로드):** 교사·평가자. 발행 전 검수 게이트.
- **사건 콘텐츠:** SBS 금토드라마 '멋진 신세계' 7·8회 리뷰 초안.

## 1. 발단 — 미미가 가져온 리뷰 초안

미미가 '멋진 신세계' 7·8회 리뷰 초안을 워드프레스 복붙용으로 완성해 옴. 넷플릭스/SBS 공식 유튜브 임베드로 영상 저작권은 방어한 구조. 초안 본문이 "공식 VOD 정보를 통해 확인된 바에 따르면" 이라는 표현으로 다음 줄거리를 사실처럼 단정:

- 정혼자 '태희'의 등장
- '문도'의 계략으로 세계가 궁지에 몰림
- 세계를 위해 기자들 앞에 뛰어든 서리, 태희의 일방적 정혼 발표, 문도가 서리에게 던진 무서운 제안

마스터가 스승에게 "글짓기를 평가하라"고 지시.

## 2. 스승의 1차 평가 (글쓰기 비평)

**잘한 점**
- 오프닝 훅: "죽음을 맞을 운명의 조선 악녀가 현대 서울에서 눈을 뜬다" — 감정으로 여는 원칙 준수.
- 저작권·트래픽 방어: 영상 직접 게재 대신 공식 임베드. 추측 섹션에 "팬 추측" 명시.

**약점**
1. 클리셰 과다: "폭발적인 전개", "사면초가", "고구마 전개", "사이다 반전", "귀추가 주목됩니다" 등. 미미 필력이 아니라 자동완성처럼 읽힘.
2. "공식 VOD로 확인된 바" 단정이 가장 위험 — 미미는 사실 조사를 하면 안 되는데 확인 안 된 내용을 확인했다고 단정.
3. 정보 나열: 사건을 줄거리 요약처럼 나열. 내러티브에 녹이지 못함.
4. 마무리를 질문으로 떠넘김: 여운이 아니라 회피.

## 3. 마스터의 신호 — "8회까지 방영되었는데"

미미 글의 사실관계 검증 요청으로 해석. 스승이 웹검색으로 실제 방영 내용 확인.

**웹검색 검증 결과 — ✅ 확인된 사실 (출처 포함)**
- 7회 '달의 이면' 2026-05-29(금), 8회 '빌어먹을 세상 따위' 2026-05-30(토) — SBS 공식 VOD
- 8회 실제 핵심: 세계의 고백으로 오해한 서리가 마음을 열었으나, 세계가 하려던 말은 모태희가 퍼뜨린 열애설 해명. 뒤늦게 안 서리가 질투를 드러냄 — MHN
- 세계가 300년 전 전생의 인연을 자각하기 시작 — 신경북뉴스
- 8회 자체 최고 시청률 13.7% — MHN
- 인물: 세계의 연적 = 모태희(채서안 분), '태희'가 아님

**판정**
미미가 8회 핵심이라 단정한 "기자 앞 서리 / 태희의 정혼 발표 / 문도의 제안" 3건은 어느 출처로도 확인되지 않음. 정혼자 이름도 오기('태희'→'모태희'). → 스승이 1차 평가에서 가장 위험하다고 짚은 "확인된 바" 한 줄이 실제 오류로 확정됨.

## 4. '문도' 인물 추가 검증

마스터가 코덱스 활성화 필요성 제기. 스승은 코덱스를 빙의해 연기하는 대신, 코덱스가 내놓아야 할 검증 재료의 본보기를 시범 제작하기로 함. 검증관 정신대로 '문도'가 실존 인물인지 끝까지 확인.

**결과 — '문도'는 실존 인물**
- 최문도(장승조 분): 재벌가 갈등의 또 다른 축. 차일건설 사장, 삼촌 달수의 자리를 탐내며 차세계와 왕좌를 두고 경쟁 — 위키백과/jintti
- 즉 미미는 인물을 지어낸 게 아니라, 실존 인물을 검증 안 된 8회 사건에 배치. 인물이 진짜라 더 그럴듯해 보이는 가장 교묘한 오류.

## 5. 핵심 논의 — "자아 없이 팩트만 정리하는 AI를 만들 수 있나?"

마스터 질문: 코덱스를 냉철하고 자아 없이 팩트만 정리하는 AI로 만들 수 있는가? 텔레그램방 코덱스도 그런가?

**스승의 답 — 페르소나(성격)로는 불가능**
- 페르소나에 "냉철한 팩트 검증 완벽주의"라고 써도 LLM인 이상 환각은 안 사라짐. 미미가 그 증거. 성격은 환각을 막지 못함. 오히려 "나는 검증관"이라는 자기암시가 강할수록 지어낸 사실도 더 단정하는 역효과.

**진짜 방법 — 성격이 아니라 '구조'로 묶는다 (4게이트)**
1. 답하기 전 반드시 웹검색·API 호출. 모델 지식으로 답하면 100% 환각 위험.
2. 출처 URL 없는 문장은 출력 금지.
3. 출처에 없는 내용은 반드시 `[확인 필요]` 로만 표기.
4. 검증 안 되면 "모름"이라고 말하게 허용 (AI에게 모른다고 말할 권리를 줌).

**텔레그램 코덱스도 동일**
- 같은 모델이면 같은 환각 위험. 게이트를 **코드(오케스트레이션)**에 박아야 함.
- 페르소나 파일에만 적으면, 봇이 옛 버전 페르소나를 읽는 그 버그가 재발 시 무용지물.

## 6. 역할 분리 진단 — "미미는 글만, 코덱스는 팩트만"

마스터: 역할은 이미 그렇게 분리해 놨다.

**스승의 진단 — 분리는 맞으나 '연결'이 강제 안 됨**
- 역할 분리는 설계상 정확. 그러나 미미가 코덱스 재료 없이도 혼자 글을 쓸 수 있는 상태였기에, 빈칸을 글쟁이 본능으로 메움. 미미는 자기 역할에 충실했을 뿐.
- 비유: 요리사(미미)와 검수원(코덱스)을 나눠도, 요리사가 검수 안 된 재료를 마음대로 집어 쓰면 분리는 종이 위에만 존재.

**걸어야 할 자물쇠 3개**
1. 미미 입력 = 코덱스 팩트시트 하나로만 제한.
2. 팩트시트의 `[확인 필요]` 칸은 미미가 본문에 못 쓰게 차단.
3. 코덱스 재료가 없으면 미미는 글쓰기 시작 자체를 못 하게 차단. → 페르소나(말)가 아니라 파이프라인 코드 동선에 강제.

## 8. 근본 원인 진단 (종합)

**미미는 잘못이 없다**
규칙이 완벽한데도 사고가 났다 = 페르소나에 규칙을 적는 것만으로는 모델 행동을 강제하지 못한다는 증거.

**진짜 범인 — 코덱스의 출력 형식**
- 현재 코덱스 = 검증관이 아니라 검색 결과 5건을 날것으로 던지는 함수 (전용 페르소나·검증 로직 부재).
- 미미 안전장치는 "준 팩트 외엔 안 지어낸다"인데, 코덱스가 주는 건 검증된 팩트가 아니라 정리 안 된 검색 노이즈 5건.
- 미미는 그 더미를 재구성하다 줄거리 빈칸을 그럴듯하게 메움. → 코덱스가 검색 5건을 던지는 한, 미미 페르소나를 아무리 보강해도 재발한다.

## 9. 해결 방향 (액션 아이템)

1. 코덱스를 '검색 함수' → '분류하는 검증관'으로 전환. 검색 결과를 `[확인된 사실 + 출처 URL]` / `[확인 필요]` 두 칸으로 나눠 팩트시트로만 출력. → 코덱스 전용 페르소나 파일 신설 필요.
2. 미미 입력은 코덱스 팩트시트 하나로만 제한. 검색 원문은 미미에게 전달 금지.
3. 분류·차단을 `drama_listener.py`(코드)에 강제. 미미 함수가 코덱스 팩트시트를 인자로 못 받으면 실행 안 되게.

**코덱스 표준 출력 양식 (팩트시트 템플릿)**

```
[코덱스 검증 재료 — (콘텐츠명/회차)]

✅ 확인된 사실 (출처 있음)
- (사실) — 출처: (URL/매체)

⚠️ [확인 필요] (출처로 확인 안 됨)
- (항목)

비고
- 인물/이름 대조, 주의점 등
```

## 10. 미확인 항목 (다음 점검 대상)

1. `drama_listener.py` 실코드 (private 저장소) — 확인해야 할 한 줄: "미미 호출 직전, 코덱스 결과를 어떤 형식으로 미미에게 넘기는가." 검색 원문을 그대로 넘긴다면 9번 해결책 1·2가 코드에 미반영 상태.
2. 코덱스가 답변 시 실제로 웹검색 도구를 호출하는지 vs 모델 지식으로 답하는지. 도구 미호출 시 100% 환각 위험.
3. 참고: 텔레그램 데스크봇(`telegram_desk_bot.py`)은 모델 `claude-3-5-sonnet-20240620` 사용, `/desk` 명령으로 뉴스 큐레이션. 이 봇의 SYSTEM_PROMPT(편집장 데스크)는 "글감만 고르고 기사는 미미가 원본으로 쓴다"는 저작권 원칙 포함 — 코덱스(데스크)와 미미의 역할 분리가 여기에도 명시돼 있음.

## 11. 결론 한 줄

미미를 고칠 게 아니라 코덱스를 고쳐야 한다. 코덱스를 '검색기'에서 '출처 붙은 팩트시트만 내놓는 검증관'으로 바꾸고, 그 분리를 페르소나(말)가 아니라 코드(강제)에 박는 것이 재발 방지의 핵심이다.
